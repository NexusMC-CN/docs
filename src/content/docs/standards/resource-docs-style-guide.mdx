## 文档目标

资源说明不是宣传页。它首先要帮助用户判断、安装、配置、使用和排错。

::: important
资源正文必须在站内自洽。外部官网、仓库或群公告可以作为补充，但不能替代站内说明。
:::

一篇合格资源说明至少回答这些问题：

- 这是什么。
- 适合谁使用。
- 支持哪些版本。
- 怎么安装。
- 有哪些依赖。
- 怎么配置。
- 常见问题怎么解决。
- 更新后需要注意什么。

## 推荐结构

### 简介

用 1 到 3 段说明资源用途、适用场景和核心特点。

推荐：

```md
这是一个面向生存服的轻量经济菜单插件，适合需要快速搭建商店、传送和公告入口的小型服务器。
```

不推荐：

```md
很好用，功能很多，自己下载看。
```

### 兼容范围

写清楚平台、游戏版本、加载器或服务端核心。

```md
- Minecraft Java：1.20.1
- 服务端核心：Paper 1.20.1
- Java：17+
- 数据库：SQLite / MySQL
```

### 安装

安装步骤必须能让第一次使用的用户照着完成。

```md
1. 下载 `ExamplePlugin.jar`。
2. 将文件放入服务器的 `plugins/` 目录。
3. 重启服务器。
4. 在 `plugins/ExamplePlugin/config.yml` 中修改数据库配置。
5. 执行 `/example reload` 重新加载配置。
```

### 配置

列出关键配置项，不要整段粘贴巨大配置文件。配置项应说明默认值、含义和何时修改。

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| `database.type` | `sqlite` | 数据库类型。小型服务器可用 SQLite。 |
| `menu.main-title` | `主菜单` | 主菜单标题。 |

### 使用方式

给出最短可用流程，让用户知道安装后第一步做什么。

```md
1. 管理员执行 `/example setup` 初始化菜单。
2. 玩家执行 `/menu` 打开主菜单。
3. 管理员通过 `/example reload` 热重载配置。
```

### 权限与命令

插件、服务端工具和管理类资源应列出命令和权限。

| 命令 | 权限 | 说明 |
| --- | --- | --- |
| `/menu` | 无 | 打开主菜单。 |
| `/example reload` | `example.admin` | 重载配置。 |

### 排错

排错章节要写“现象、原因、处理方式”，不要只写“报错请加群”。

| 现象 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 插件未加载 | Java 版本过低。 | 升级到 Java 17 或更高。 |
| 菜单打不开 | 权限插件未授权。 | 给玩家组添加 `example.menu`。 |

### 更新说明

长期维护资源应保留更新说明，按版本记录重要变化。

```md
## 更新说明

### v1.2.0

- 新增 MySQL 支持。
- 修复首次启动时配置文件未生成的问题。
- 调整默认菜单标题。
```

## 写作要求

- 一段只表达一个重点。
- 命令、路径、配置项、文件名用行内代码。
- 步骤使用有序列表。
- 参数说明使用表格。
- 截图要说明它展示的功能或状态。
- 不要把安装、配置、更新说明混在同一段里。
- 不要把核心信息藏在图片里，文字仍要写清楚。

## 语气要求

资源说明应清楚、克制、可验证。

推荐：

- “已在 Paper 1.20.1 测试。”
- “需要 Java 17 或更高版本。”
- “如果从 1.1.x 升级，需要删除旧配置中的 `legacy-mode`。”

不推荐：

- “全网最好。”
- “无敌稳定。”
- “应该都能用。”
- “不会的自己研究。”

## 常见反例

### 只有宣传，没有使用说明

```md
超强插件，功能丰富，稳定高效，欢迎下载。
```

问题：用户不知道怎么安装、怎么配置、适配什么版本。

### 只有更新，没有当前说明

```md
v1.0 修复 bug
v1.1 优化体验
v1.2 增加功能
```

问题：新用户不知道资源当前能做什么。

### 只给外链，不写正文

```md
详情看官网：https://example.com
```

问题：站内搜索、审核和用户快速判断都失效。外链可以作为补充，但站内正文必须完整。

::: warning
只给外链不写正文，会让用户无法在站内判断资源是否适合自己，也会让审核人员无法确认内容质量。
:::

## 提交前检查清单

::: details 展开检查清单

- 用户能否只看正文就完成安装。
- 版本、依赖、平台是否写清楚。
- 下载方式是否与正文一致。
- 配置项是否解释了用途。
- 排错信息是否包含处理方式。
- 更新说明是否能帮助老用户判断是否升级。

:::

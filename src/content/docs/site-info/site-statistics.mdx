## 统计口径

主站统计分为公开统计、后台统计和个人统计三类。不同统计面向不同用户，更新频率和可见范围也不同。

::: note
统计数字必须结合口径理解。公开统计、后台统计和个人统计即使名字相似，也可能不使用同一套筛选条件。
:::

| 类型 | 面向对象 | 常见用途 |
| --- | --- | --- |
| 公开统计 | 所有访问者 | 展示社区规模和当前活跃情况。 |
| 后台统计 | 管理员和具备后台权限的角色 | 了解站点内容、审核、举报、用户和系统运行情况。 |
| 个人统计 | 登录用户本人和访问其主页的用户 | 展示用户贡献、发帖、资源、回复和活动记录。 |

## 首页社区统计

首页社区统计来自公开接口，主要展示：

- 资源总数：已通过审核并公开的资源数量。
- 帖子总数：公开可见的帖子数量。
- 今日新发：当天新增的公开帖子和公开资源数量。
- 在线人数：当前在线连接统计。

这些数据有缓存，不保证秒级实时。短时间内刚发布的内容没有立刻反映到统计卡片中，通常是缓存尚未刷新。

::: tip
刚发布或刚审核通过的内容，等待缓存刷新后再检查首页统计会更可靠。
:::

## 后台统计

后台统计用于站点维护，不一定对普通用户公开。常见统计项包括：

- 用户总数、管理员数量、封禁用户数量。
- 资源总数、已公开资源、待审核资源、下载总量。
- 帖子总数、已公开帖子、待审核帖子。
- 回复、评论、点赞等互动数量。
- 私信总量和被风控标记的私信数量。
- 举报总量和待处理举报数量。
- 资源版本、板块、分类、标签等基础数据数量。
- 备份和系统维护相关记录。

后台统计更适合管理员判断站点运行状态，不适合直接当作公开排行榜。

::: important
后台统计可能包含待审核、隐藏、锁定或维护数据，不适合作为公开展示口径直接引用。
:::

## 用户个人统计

用户个人页和个人中心通常会展示与该用户相关的数据，例如：

- 总活跃度。
- 发帖数量。
- 资源数量。
- 回复或评论活动。
- 收藏、历史、关注或其他个人记录。
- 签到情况和积分流水。

个人统计会受到隐私设置、内容状态和访问权限影响。别人查看你的主页时，不一定能看到你本人在个人中心看到的全部数据。

## 统计为什么会不一致

不同页面的统计不一致通常有这些原因：

- 统计口径不同：公开统计只算公开内容，后台统计可能包含待审核或被隐藏内容。
- 缓存时间不同：公开统计为了性能会缓存，后台或个人页可能实时性更高。
- 权限不同：你只能看到自己有权限查看的内容。
- 内容状态变化：草稿、待审核、拒绝、隐藏、锁定会影响是否计入公开统计。
- 数据回填或迁移：历史内容在补齐字段时，统计可能短时间内变化。

::: details 快速排查统计不一致

1. 先确认两个页面统计的对象是否相同。
2. 再确认是否只统计公开内容。
3. 检查内容是否处于 `draft`、`pending`、`rejected`、隐藏或锁定状态。
4. 等待缓存刷新后再比较。
5. 如果仍明显异常，再联系管理员。

:::

## 统计数据能否手动修正

普通用户不能直接修改统计数据。若发现明显异常，可以联系管理员，并提供页面链接、截图、时间和你认为异常的统计项。

管理员可以根据后台记录、活动流水、内容状态和系统日志判断是否需要修复。

## 等级是什么

用户等级是主站根据活跃度展示的身份标识。它主要反映用户在站点里的持续参与程度，不等同于管理权限。

等级由用户的 `contribution` 活跃度计算。活跃度越高，等级越高；被封禁或暂停的用户会显示特殊状态等级。

::: important
用户等级不是管理权限。等级提高不会自动获得管理员、总版主或分版主能力。
:::

| Lv0 | 洞穴蜘蛛 | 0 |
| Lv1 | 草原兔 | 10 |
| Lv2 | 哞菇牛 | 50 |
| Lv3 | 草原狼 | 150 |
| Lv4 | 末影人 | 400 |

实际阈值可以在后台配置中调整。文档中的数值说明默认口径，页面显示以当前站点配置为准。

::: note
如果页面显示的等级阈值和本文不同，以当前站点后台配置为准。
:::

## 活跃度来源

常见活跃度来源包括：

- 每日签到。
- 发布帖子。
- 发布资源。
- 回复帖子。
- 评论资源或其他内容。
- 管理员因活动、补偿或人工修正进行调整。

默认奖励值可由后台配置调整。当前实现里常见默认值是：签到 `+1`、发帖 `+2`、发资源 `+2`、回复 `+1`、评论 `+1`。

::: details 默认活跃度来源参考

| 行为 | 常见默认奖励 |
| --- | --- |
| 签到 | `+1` |
| 发帖 | `+2` |
| 发资源 | `+2` |
| 回复 | `+1` |
| 评论 | `+1` |

:::

## 签到与连续签到

签到是最稳定的活跃度来源。

签到时系统会：

1. 检查当天是否已经签到。
2. 计算连续签到天数。
3. 发放基础活跃度奖励。
4. 每连续 7 天给一次额外加成，额外加成有上限。

如果你购买或使用补签类道具，是否补发活跃度取决于该道具配置。

## 等级有什么用

等级主要用于展示用户活跃程度，也可能被部分功能作为门槛参考。例如组织相关能力可以配置最低等级要求。

等级不会自动授予管理员、总版主或分版主权限。管理权限必须由管理员单独分配。

## 为什么等级没变

等级没有变化通常有几种原因：

- 当前活跃度还没有达到下一等级阈值。
- 页面缓存尚未刷新。
- 活跃行为没有触发奖励，例如重复签到、内容未成功发布或操作失败。
- 管理员调整了等级阈值，导致所需活跃度变化。
- 用户处于封禁或暂停状态，等级显示被特殊状态覆盖。

如果你确认活跃度已经变化但页面仍旧不更新，可以刷新页面或稍后再看。

::: tip
等级显示可能受缓存影响。刚完成签到、发帖或资源发布后，稍后刷新页面再确认会更准确。
:::

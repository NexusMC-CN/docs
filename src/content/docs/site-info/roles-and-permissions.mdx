## 角色总览

主站角色用于区分用户在站点里的管理范围。普通用户主要使用发帖、投稿、评论、收藏、私信等功能；管理角色会额外拥有审核、处理举报、维护内容秩序等能力。

::: note
角色说明用于理解权限边界，实际可操作范围仍以当前后台配置和分配范围为准。
:::

| 角色标识 | 前台名称 | 权限范围 |
| --- | --- | --- |
| `admin` | 管理员 | 站点级管理权限，覆盖内容审核、用户管理、站点配置和管理后台。 |
| `moderator` | 总版主 | 全站内容管理与审核权限，覆盖帖子、资源、找服玩、视频和举报处理。 |
| `forum_mod` | 论坛版主 | 管理被分配的论坛板块内帖子、回复和相关举报。 |
| `resource_mod` | 资源版主 | 管理被分配的资源分类内资源、资源版本、资源评论和相关举报。 |
| `player_mod` | 找服玩版主 | 管理找服玩服务器投稿、评论和相关举报。 |
| `video_mod` | 视频版主 | 管理被分配的视频分区内视频、回复和相关举报。 |
| `user` | 会员 | 使用主站普通功能，不具备后台审核权限。 |

## 管理员

管理员是站点最高管理角色，通常负责站点配置、用户状态、角色分配、内容分类、审核策略和系统级维护。

管理员可以处理：

- 用户账号、角色、封禁和修复。
- 帖子、资源、找服玩、视频的审核与管理。
- 板块、分类、标签、通知模板等站点配置。
- 举报、风控、SEO、存储、邮件、维护模式等后台设置。

管理员操作会记录后台日志，用于追踪谁在什么时候改动了什么内容。

::: caution
涉及账号安全、系统配置、密钥、存储、邮件和角色分配的操作，应只由管理员处理。
:::

## 总版主

总版主对应 `moderator` 角色，偏向全站内容秩序维护。它拥有全局审核和内容管理能力，但不等同于管理员。

总版主通常可以处理：

- 所有板块的帖子和回复。
- 所有资源分类的资源、版本和评论。
- 找服玩服务器投稿和评论。
- 视频投稿、视频回复和相关举报。
- 用户违规处理，例如封禁、警告或协助管理员处理争议。

总版主不应默认承担站点配置、系统密钥、存储、邮件、SEO、角色分配等系统管理职责。

::: important
总版主是全站内容管理角色，不等同于管理员。它负责内容秩序，不负责系统级配置。
:::

## 分版主

分版主只负责某一类内容或某些具体范围。分版主的权限比总版主更窄，适合按板块、分类或业务线分工。

| 分版主类型 | 主要负责内容 | 范围控制 |
| --- | --- | --- |
| 论坛版主 | 帖子、回复、帖子评论、帖子举报 | 可按论坛板块分配范围。 |
| 资源版主 | 资源、资源版本、资源评论、资源举报 | 可按资源分类分配范围。 |
| 找服玩版主 | 找服玩服务器投稿、评论、举报 | 当前按找服玩业务线管理。 |
| 视频版主 | 视频投稿、视频回复、视频举报 | 可按视频分区分配范围。 |

分版主应优先处理自己负责范围内的内容，不应越权处理与自己无关的业务线。

::: tip
如果一个问题只发生在某个板块、资源分类或视频分区，优先找对应分版主会更快。
:::

## 权限边界

管理权限的基本原则是最小必要权限。

- 能用分版主解决的，不直接给总版主。
- 能用总版主解决的，不直接给管理员。
- 临时协助处理事务时，应在事务结束后回收临时权限。
- 涉及用户账号、密码、安全验证、系统设置和站点配置的事项，应交由管理员处理。

::: details 权限分配参考

- 日常内容审核：优先分配给对应分版主。
- 跨分区内容争议：可交由总版主协调。
- 账号、安全、系统配置：交由管理员处理。
- 临时活动或专项整理：可以临时授权，完成后回收。

:::

## 用户能看到什么

普通用户在前台通常能看到用户昵称旁的角色标识，例如管理员、版主、资源版主等。角色标识用于帮助用户判断回复或审核意见的来源，但不代表该用户可以代表站点做所有决定。

如果同一个问题涉及多个分区，优先按内容所在位置联系对应版主；如果无法判断，联系总版主或管理员。

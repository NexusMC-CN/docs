## 适用范围

这篇文档适用于资源区内容，包括材质、模组、插件、数据包、整合包、地图、服务端相关资源等。

资源投稿比普通帖子更严格，因为它会影响下载安全、版本兼容、授权归属和后续维护。

::: important
资源投稿的重点不是“填完表单”，而是让用户能判断资源是否安全、可用、可维护。
:::

## 状态流转

| 状态 | 说明 | 用户能做什么 |
| --- | --- | --- |
| `draft` | 草稿，仅作者可见。 | 补齐信息，继续编辑或提交审核。 |
| `pending` | 已提交审核。 | 等待审核结果。 |
| `approved` | 审核通过并公开展示。 | 可继续维护；更新后可能重新审核。 |
| `rejected` | 审核未通过。 | 根据驳回原因修改后重新提交。 |

## 标准流程

1. 创建资源草稿。
2. 填写标题、分类、正文、版本、兼容版本、下载方式等信息。
3. 上传或填写封面、图集、下载链接和补充链接。
4. 保存草稿或提交审核。
5. 服务端检查字段、分类、正文长度、官方标签、付费规则、发布频率和增强认证要求。
6. 正式提交后进入 `pending`。
7. 审核通过后变为 `approved`；未通过则变为 `rejected`。
8. 作者后续编辑公开资源时，普通用户的重大更新通常会重新进入审核。

## 必填信息

| 信息 | 说明 |
| --- | --- |
| 标题 | 清楚表达资源名称，不要堆关键词。 |
| 分类 | 必须选择正确资源分类。 |
| 正文 | 正式提交时需要满足资源正文长度规则。 |
| 资源版本 | 建议始终填写，方便用户判断是否需要更新。 |
| 兼容版本 | 说明适配的 Minecraft 或平台版本。 |
| 下载信息 | 本地文件或外部下载链接，必须真实有效。 |

## 推荐补充信息

| 信息 | 价值 |
| --- | --- |
| 简介 | 帮助列表页快速判断资源用途。 |
| 封面 | 提升识别度，必须和资源内容相关。 |
| 图集 | 展示实际效果、配置界面或使用场景。 |
| 依赖资源 | 避免用户安装后无法运行。 |
| 教程帖 | 承载长步骤、复杂配置和演示内容。 |
| 文档引用 | 连接站内公开文档帖。 |
| 仓库地址 | 开源资源建议填写。 |
| 交流地址 | 方便用户反馈问题。 |

## 下载方式

资源支持本地下载和外部链接两类处理。

外部链接需要注意：

- `fileUrl` 必须是有效链接。
- 网盘资源可以填写 `extractCode`。
- 不要填写需要特殊权限、私密访问或随时失效的链接。

::: warning
外部链接如果需要私聊、加群、登录陌生站点或临时授权才能下载，通常不适合作为唯一下载方式。
:::

付费资源有额外限制：

- 付费资源仅支持外部链接下载。
- `fileUrl` 必须是有效的 `http` 或 `https` 地址。
- 资源所在分类必须允许发布付费资源。

::: caution[付费资源]
不要用免费资源标题吸引用户，再在正文或下载链接里临时要求付费。付费内容、授权范围和售后方式必须提前写清楚。
:::

## 草稿与正式提交

草稿适合先保存复杂资源信息。草稿阶段不会执行全部正式发布检查。

正式提交会额外检查：

- 正文最低长度。
- 官方标签是否属于当前分类允许范围。
- 付费资源规则。
- 文档引用是否来自符合要求的公开文档帖。
- 账号是否满足增强认证要求。
- 发布频率是否触发限制。

## 编辑与重新审核

公开资源不是不能改，但要避免“审核通过后完全换内容”。

普通用户更新 `approved` 或 `rejected` 资源时，通常会重新进入 `pending`。常见触发更新审核的内容包括：

- 标题、正文或简介大幅变化。
- 分类、版本、兼容范围变化。
- 下载链接或文件信息变化。
- 付费状态变化。
- 封面、图集或外部链接变化。

## 常见失败原因

::: details 展开查看资源投稿失败原因

| 场景 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 正文太短 | 正式提交时资源正文未达到最低长度。 | 补充介绍、安装、配置、使用和排错说明。 |
| 官方标签报错 | 选择了当前分类不允许的官方标签。 | 按当前分类重新选择标签。 |
| 付费资源失败 | 不是外部下载、链接无效或分类不允许付费。 | 改为有效外链，并确认分类规则。 |
| 文档引用失败 | 引用帖不是公开、已审核、无访问限制的文档帖。 | 更换符合要求的站内文档帖。 |
| 个人 Token 组织投稿失败 | 请求中传入了 `organizationId`。 | 移除组织身份字段。 |
| 触发安全设置要求 | 账号未绑定 2FA 或通行密钥。 | 先完成增强认证绑定。 |

:::

## 投稿前检查清单

::: details 展开检查清单

- 标题、简介、分类是否准确。
- 正文是否包含安装、配置、使用、排错和更新说明。
- 下载链接是否可访问。
- 版本号和兼容版本是否明确。
- 封面、图集是否展示真实资源。
- 依赖、授权、来源和付费信息是否说清楚。

:::

## 适用范围

这篇文档适用于论坛普通帖子、说明帖、更新日志、教程帖，以及资源文档中引用的公开说明内容。

帖子投稿既可以从前台表单发起，也可以通过用户个人 API 发起。无论入口不同，最终都以服务端校验结果为准。

::: tip
如果只是暂存内容，优先保存为草稿；如果希望公开展示，再提交审核。
:::

## 状态流转

| 状态 | 说明 | 用户能做什么 |
| --- | --- | --- |
| `draft` | 草稿，仅作者自己可见。 | 继续编辑，或提交审核。 |
| `pending` | 已提交，等待审核或自动审核流程处理。 | 等待结果，必要时联系管理组。 |
| `approved` | 已审核通过并公开展示。 | 可继续编辑；大幅修改后可能重新进入审核。 |
| `rejected` | 审核未通过。 | 查看驳回原因，修改后重新提交。 |

::: important
不要在文档、脚本或前台逻辑里使用 `published` 表示公开状态；当前帖子公开状态以 `approved` 为准。
:::

## 标准流程

1. 用户进入发帖页或调用接口。
2. 填写标题、正文、板块、标签、可见性等信息。
3. 选择保存草稿或提交审核。
4. 服务端检查字段格式、板块权限、正文长度、频率限制和安全策略。
5. 草稿保存为 `draft`，正式提交进入 `pending`。
6. 审核通过后变为 `approved`；未通过则变为 `rejected` 并记录原因。
7. 作者可继续编辑自己的帖子；编辑已公开或已驳回内容时，通常会重新进入 `pending`。

## 必填字段

| 字段 | 说明 |
| --- | --- |
| 标题 | 必须有明确主题，不要只写“求助”“看看”。 |
| 正文 | 正式提交时需要满足站点正文长度规则。 |
| 板块 | 必须选择允许当前用户发帖的板块。 |

## 可选字段

| 字段 | 用途 |
| --- | --- |
| 标签 | 帮助检索和推荐，最多 20 个。 |
| 话题标签 | 按板块允许范围选择，不能跨板块乱选。 |
| 可见性 | 可配置公开、部分可见等模式。 |
| 访问密码 | 密码访问时必须填写 4 到 64 字符的密码。 |
| 自定义路径 | 用于生成更稳定、可读的公开链接。 |
| 版权声明 | 适合教程、原创文章和长期维护内容。 |

## 草稿与正式提交

草稿适合未写完、需要临时保存的内容。草稿不会进入公开列表，也不会触发完整的正式投稿检查。

正式提交会触发更多规则：

- 正文长度检查
- 板块发帖权限检查
- 增强认证策略检查
- 内容发布频率限制
- 内容安全与审核流程

因此“草稿能保存”不代表“正式投稿一定能通过”。

::: warning
草稿只代表数据暂存成功。正式提交时仍可能因为正文长度、板块权限、安全策略或频率限制失败。
:::

## 编辑规则

作者可以编辑自己的帖子。编辑时注意：

- 修改草稿并提交时，状态会从 `draft` 进入 `pending`。
- 修改 `approved` 或 `rejected` 帖子后，普通用户通常会重新进入审核。
- 如果帖子属于部分可见，必须保证允许查看用户列表有效。
- 如果帖子使用密码访问，必须保证访问密码符合长度要求。
- 不要通过编辑正文无痕覆盖已被举报或已被管理组处理的问题内容。

::: caution
已被举报、锁定或管理组处理过的问题内容，不应通过普通编辑绕过处理记录。
:::

## 审核通过建议

更容易通过审核的帖子通常具备这些特征：

- 标题能说明具体主题。
- 正文包含背景、过程、结果或明确问题。
- 所属板块和内容主题一致。
- 标签克制且准确。
- 没有外链诱导、搬运争议或不完整说明。

## 常见失败原因

::: details 展开查看帖子投稿失败原因

| 场景 | 可能原因 | 处理方式 |
| --- | --- | --- |
| 提交后返回“请选择板块” | `boardId` 为空或板块不存在。 | 重新选择正确板块。 |
| 提示正文太短 | 正文未达到正式投稿最低长度。 | 补充背景、步骤、截图说明或结论。 |
| 提示无权在板块发帖 | 当前账号没有该板块发帖权限。 | 换到允许发帖的板块，或联系管理组。 |
| `visibilityAllowUserIds is required` | 部分可见模式没有指定允许查看用户。 | 补齐用户 ID 列表，或改为公开可见。 |
| `accessPassword is required` | 选择了密码访问但没有填写密码。 | 填写 4 到 64 字符的访问密码。 |
| `SECURITY_SETUP_REQUIRED` | 站点要求先绑定 2FA 或通行密钥。 | 到账号设置中完成增强认证。 |
| `429` | 发布频率过高或触发内容限制。 | 降低提交频率，稍后重试。 |

:::

## 投稿前检查清单

::: details 展开检查清单

- 标题是否能让用户一眼知道内容主题。
- 正文是否包含足够上下文，而不是只贴一句话。
- 板块是否准确。
- 标签是否真实描述内容。
- 如果是教程，是否包含环境、步骤和预期结果。
- 如果是反馈或求助，是否包含复现方式、报错信息和已尝试的方法。

:::

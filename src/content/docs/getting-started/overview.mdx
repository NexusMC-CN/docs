## 这是什么

这是 NexusMC 的帮助中心和文档入口，用来集中放置投稿流程、资源规范、用户个人 API 和后续站点规则。

如果你只是想完成一件具体事情，不需要从头读完整个文档中心。先按自己的目标选择路径即可。

::: tip
第一次来文档中心时，先按目标跳转，不需要按左侧目录从上到下阅读。
:::

## 文档怎么分类

当前文档分为五类：

| 分类 | 内容 |
| --- | --- |
| 快速开始 | 帮助第一次进入文档中心的人找到入口，也说明文档维护方式。 |
| API | 面向脚本、自动化工具和开发接入。 |
| 投稿流程 | 说明帖子、资源、找服玩、视频从草稿到审核公开的流程。 |
| 资源规范 | 说明资源首发、更新和正文编写要求。 |
| 主站信息 | 说明站点角色、用户等级、常见问题和统计口径。 |

## 投稿相关状态

很多投稿文档都会提到状态。当前主要状态如下：

| 状态 | 含义 |
| --- | --- |
| `draft` | 草稿，仅作者可见。 |
| `pending` | 已提交，等待审核。 |
| `approved` | 审核通过，公开展示。 |
| `rejected` | 审核未通过，需要修改后重新提交。 |

如果你看到旧资料里写 `published`，在当前文档语境中应理解为 `approved`。

::: important
当前文档统一使用 `approved` 表示已审核并公开展示。脚本、文档和前台文案都不应混用 `published`。
:::

## 推荐阅读顺序

如果你是普通用户：

::: details 普通用户阅读路径

1. 先读与你要投稿的内容类型对应的“投稿流程”。
2. 如果是资源，再读“资源规范”。
3. 想了解账号、等级、版主或统计口径时，再看“主站信息”。
4. 遇到报错时，先看对应流程文档里的“常见失败原因”。

:::

如果你是脚本或工具开发者：

::: details 开发者阅读路径

1. 先读“用户个人 API”。
2. 确认需要的 scope。
3. 再回到对应投稿流程，理解状态和审核规则。

:::

如果你是文档维护者：

::: details 文档维护者阅读路径

1. 先读“使用 MDX 编写文档”。
2. 新增内容时先选好分类和排序。
3. 写完后本地构建确认页面能生成。

:::

## 文档更新原则

文档应以当前服务端真实实现为准。不要把规划中的能力写成已经开放的能力。

::: warning
不要把“计划支持”“后续可能开放”的能力写进用户操作流程。用户文档只写当前可用、可验证的行为。
:::

写文档时优先回答：

- 用户现在能做什么。
- 需要准备什么。
- 失败时会看到什么。
- 应该怎么修正。
- 哪些能力当前不支持。
